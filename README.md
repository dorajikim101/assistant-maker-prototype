# OpenClaw Fairy Maker Prototype

A lightweight Princess-Maker-inspired onboarding prototype for a persistent AI assistant.

## Concept
- Start from a **small fairy assistant**
- Let the user shape it via **name, tone, books, choices, and training**
- Update the assistant's **form and role** over time
- End with a usable **working persona / export profile**

## Direction
This prototype intentionally borrows the **raising-game structure** but uses an original visual direction:
- library fairy / record spirit as the default route
- optional growth into office, mechanic, or weird custom forms
- trainer-AI concept for domain-specific education
- static UI focused on validating screen composition quickly

## Included screens
- **Art Direction Board**
- **Birth / role setup**
- **Library / book-based preference inference**
- **Training / trainer-based specialization**
- **Ending / export preview**

## Current visual themes
1. **Library Fairy Assistant** — main route
2. **Mechanic Operator** — precision / automation route
3. **Weird Custom Variant** — for user-driven mutation and image generation concepts
